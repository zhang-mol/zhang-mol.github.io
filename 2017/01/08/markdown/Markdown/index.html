Markdown编辑器

国内Markdown编辑器很多,针对不同的系统开发了不同的编辑器,而且各大编辑器都没有统一的规定,会存在一些区别.为避免打广告和引发辩驳大战,不透露本人使用的IDE.Markdown支持部分html标签,为了让文本更形象,更简洁.

标题

引用

区块引用使用 >;若想结束引用,在末尾添加空白行即可.

一个人至少有一个梦想,有一个理由去坚强.心若没有栖息的地方,到哪里都是在流浪.

区块引用可以嵌套,一个 >为一层,两个>>为两层

三毛

岁月极美,在于它必然的流逝.春花、秋月、夏日、冬雪.

区块引用可以使用其他的Markdown语法,比如:标题、代码块、列表等.

列表

Markdown支持无序列表和有序列表

无序列表可以使用星号(*)、加号(+)、减号(-)

效果如下:

  • 如果有来生，
  • 要做一颗书，
  • 站成永恒，
  • 没有悲欢的姿势。
  • 一半在土里安详，
  • 一半在风里飞扬，
  • 一半洒落阴凉，
  • 一半沐浴阳光。
  • 非常沉默非常骄傲，
  • 从不依靠，从不寻找。

有序列表则是数字加一个英文句点(数字不会影响输出的html结果)

效果如下:

  1. 我来不及认真年轻
  2. 待明白过来时
  3. 只能选择认真地老去
  4. 或许，我们终究会有一天
  5. 牵着别人的手
  6. 遗忘曾经的他

列表项目可以包含空行、段落、引用、代码块

  • 包含空行,会为元素添加p标签
  • 包含段落、引用时需要缩进(4个空格)
  • 包含代码块时需要两次缩进(8个空格)
