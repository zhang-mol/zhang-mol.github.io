渐进增强与优雅降级

发表于 2017-05-13 | 分类于 Web | 阅读次数
| 642 (字) | 2 (分)

渐进增强与优雅降级是在CSS3之后活起来的，因为CSS3的特效很炫酷，而低版本的浏览器不支持CSS3，只能保证最基本的功能。而高版本的浏览器则使用CSS3增强用户体验。两者的目的都是为了增强用户体验，但是因为侧重点不同，导致工作流程也产生了不同。

渐进增强

  一开始就针对低版本浏览器进行构建页面，完成基本的功能，然后再针对高级浏览器进行效果、交互、追加功能达到更好的体验。也称之为向上兼容。

  渐进增强的写法，优先考虑老版本浏览器的可用性，最后才考虑新版本的可用性。在前缀CSS3和正常CSS3都可用的情况下，正常CSS3会覆盖前缀CSS3。

优雅降级

  一开始就构建站点的完整功能，然后针对浏览器测试和修复。比如一开始使用 CSS3 的特性构建了一个应用，然后逐步针对各大浏览器进行 hack 使其可以在低版本浏览器上正常浏览。也称之为向下兼容。

  优雅降级的写法，优先考虑新版本浏览器的可用性，最后才考虑老版本的可用性。在前缀CSS3和正常CSS3都可用的情况下，前缀CSS3会覆盖正常的CSS3。

两者比较

  就CSS3单独而言，更倾向于渐进增强，因为在CSS3某些属性在不同的浏览器可能不一样，所以最终还是以正常CSS3为准。如果你好奇究竟是什么属性在前缀CSS3和正常CSS3中显式效果不一样，可以看看这篇文章需警惕css3属性的书写顺序
  至于如何抉择？根据你的用户所使用的客户端的版本来做决定。如果低版本用户居多，当然优先采用渐进增强的开发流程；如果高版本用户居多，为了提高大多数用户的使用体验，那当然优先采用优雅降级的开发流程。
  绝大多数的大公司都是采用渐进增强的方式，因为业务优先，提升用户体验永远不会排在最前面。具体的抉择，视情况而定。
